# PROJECT: joySite (Designer Portfolio)

## Vision
A pixel-perfect, highly animated designer portfolio website that serves as a flexible template, allowing the user to easily update content without touching the core code or animation logic.

## Core Goals
1.  **Pixel Perfect:** Strictly follow the provided Figma design specs.
2.  **Animation First:** Leverage GSAP (GreenSock Animation Platform) for all complex, smooth animations.
3.  **Template Architecture:** Decouple content from layout. All text, images, and project details should be easily updatable via structured data (e.g., JSON, Markdown, or a lightweight CMS) rather than hardcoded in the layout.

## Constraints & Preferences
-   The design will be imported via Figma MCP.
-   The underlying code structure must remain clean to avoid conflicting with GSAP's DOM manipulations.
-   Animations should not break responsiveness (must work flawlessly on mobile and desktop unless specified otherwise in Figma).
